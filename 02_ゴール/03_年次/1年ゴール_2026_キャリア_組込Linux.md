# 1年ゴール_2026_キャリア_組込Linuxプロトタイピング

## 1. この1年のテーマ
組込Linux（EMLinux）を、効率的なプロトタイプ開発の手段として使える状態にする。
特に Flutter によるGUIを成立させ、ネットワーク機能とMCU(UART)連携までを一気通貫で確認する。

## 2. 現在地（2025末）
- EMLinuxサンプルが実機で動作した（as-is構成）。
- 次に検証したい要素：GUI(Flutter)、カスタムレイヤー、ネットワーク追加、UARTで別MCU連携。

## 3. 成果指標（KGI）
- KGI-1: 「EL ProtoKit v1」を完成（Flutter UI + ネットワーク + UART(MCU)連携を含むデモ）。
- KGI-2: 再現性（ビルド手順、カスタムレイヤー、起動手順、デモ実行手順）を整備し、他メンバーが再現できる。
- KGI-3: 採用判断のための提案資料（技術選定の理由、メリデメ、今後の拡張方針）を1本作る。

## 4. 行動指標（KPI）
- 月20hを確保（主に Platform/Build と統合に投資）。
- 月1回 ADR（設計判断メモ）を更新する。
- 四半期ごとに Gate をレビューし、次Q計画に反映する。

## 5. 想定するリファレンス設計（EL ProtoKit v1）
- UI: Flutter（フルスクリーン）
- Backend: UART/Networkを吸収するサービス（proto-hub）
- MCU: 既存システムのサブマイコンをUARTで接続（検証用）
- Network: SSH + 状態確認API（HTTP/MQTTは後で決める）
