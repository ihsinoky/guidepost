## 1. この1年のテーマ
Copilot（GitHub Copilot / Copilot Studio）を「個人の工夫」から「再現できる業務改善」に引き上げる。

## 2. 具体的なゴール
- GitHub Copilot：開発タスク（実装・テスト・調査・修正）の速度と品質を両立して改善する。
- Copilot Studio：特定の業務フロー（例：ドキュメント作成、レビュー補助、問い合わせ対応など）を自動化/半自動化する。

## 3. 成果指標（KGI）
- 2026年末までに、仮説検証の結果として「工数削減効果」を測定し、累積効果を説明できる状態にする。
  - 定量：月X時間相当（自分 or チーム）を“計測ログ付き”で示せる
  - ガードレール：品質/安全性の基準を落とさない（レビュー・テスト・セキュリティ運用を維持）

## 4. 行動指標（KPI）
- 仮説検証を月1本（年12本）実施する
  - うち GitHub Copilot：6本 / Copilot Studio：6本
- 仮説検証 1本の完了条件：
  1) 仮説が明文化されている
  2) Before/Afterの計測がある
  3) 採用/不採用の判断がある
  4) 再現できる成果物（設定/プロンプト/フロー）が残っている
